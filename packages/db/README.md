# @cyggie/db

Async pg-backed repositories + drizzle schema for the Cyggie cloud Postgres database (Neon).

Consumed by:
- `api-gateway/` — primary consumer (mobile + future web)
- `src/main/` desktop — Phase 2 migration to thin-client mode (V1 keeps better-sqlite3 sync wrappers in [packages/db/src/repositories/](./src/repositories/) — async conversion deferred per plan §0.4b)

## Conventions

- **Multi-tenant**: every owned table has `user_id` FK + Postgres RLS policy.
- **Sync metadata**: every owned table has a `_lamport` text column (default `'0'`).
- **Writes go through `writeWithSync(table, rowId, fields)`** — single helper that bumps `_lamport`, writes the row, and appends an `outbox` entry in one transaction. Don't write to owned tables directly.
- **Timestamps**: always `timestamp with time zone`. Default `now()` for `created_at`/`updated_at`.
- **IDs**: cuid2 (text) for entities ported from SQLite (preserves desktop IDs); `serial`/`bigserial` for gateway-new append-only tables (outbox, audit_log).
- **JSON**: `jsonb`, never `json`. Always `.default({})` or `.default([])` to avoid `null` handling everywhere.
- **FTS**: `tsvector` columns with GIN indexes. Search rewrites live in the repositories.
